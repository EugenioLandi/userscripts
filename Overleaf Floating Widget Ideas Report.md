# Overleaf floating widget idea report

## Scope
- Existing Overleaf functionality already present in the repository was excluded from the candidate set: compile timer, GitHub shortcut, ZIP download, and the current notes/checklist widgets.
- Ranking criteria: **usefulness**, **practicality**, **ease of implementation**, and **differentiation** (1-10 each).
- Total score is the sum of the four criteria and is used for ranking.

## Top 5 selected for implementation
1. **Overleaf Symbol Palette** — highest overall balance of day-to-day usefulness, speed, and low-risk implementation.
2. **Overleaf Table Builder** — frequently needed structure with straightforward inputs and highly copyable output.
3. **Overleaf Figure Builder** — practical for most papers and presentations while remaining simple and robust.
4. **Overleaf Citation Helper** — useful for citation-heavy writing and easy to keep lightweight.
5. **Overleaf Color Palette** — broadly helpful for annotations/slides and visually well-suited to a floating widget.

## Notes on implementation strategy
- Each selected script is implemented as a standalone top-level userscript.
- Each one uses the same interaction model: a single button in the Overleaf sidebar that opens a draggable, resizable floating widget attached to `document.body`.
- Widget chrome uses the same less-transparent glass styling update requested for the notes/checklist script.
